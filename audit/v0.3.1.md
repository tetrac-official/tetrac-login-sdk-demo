# SDK VAULT SINGLETON

## client and react use the same vault

- all mutable vault state (key, lock deadline, timer, config, listeners, clear-hooks) moved onto the non-enumerable Symbol.for("tetrac.vault") global slot, memory-only, with a configurable slot

- every vault operation (armAppKey, touchVault, lockVault, isLocked, scheduleLock, cross-tab handlers) consistently uses the shared vault.\* global slot. The SDK side is done and shipped in 0.3.1

- arming /client makes /react's useSigner() report unlocked=true, and locking /client flips it back to false — both bundles now share one Symbol.for("tetrac.vault") vault.
